Stories will be formatted as a JSON object with the following fields:

{
    "id": "story_id",
    "userId": ["user_id_1"],
    "title": "story_title",
    "description": "story_description",
    "images": ["image_url_1", "image_url_2", ...],
    "location": "story_location",
    "date": "story_date",
    "audio": ["audio_url_1", "audio_url_2, ...],
    "content": "
    {
        "text": "story_text in the beginning",
        "images": ["image_url_1"],
        "text": "story_text in the middle",
        "videos": ["video_url_1"],
        "text": "story_text in the middle",
        "images": ["image_url_3"],
        "text": "story_text in the end",
    }",
    "createdAt": "timestamp",
    "updatedAt": "timestamp"
}

The content field is a JSON object with the following fields:

{
    "text": "story_text",
    "images": ["image_url_1", "image_url_2", ...],
    "videos": ["video_url_1", "video_url_2", ...],
}

Where the user can add text, images, videos, and audio to the story and it will be displayed in the order they are added.
This allows the user to create a story with a more natural flow and feel.

The images, videos, and audio are optional and can be added to the story at any point.
The text is required and must be added to the story.

The images, videos, and audio are stored in the Firebase Storage and the URLs are stored in the Firestore database.

The story is created by the user and the userId is the id of the user who created the story.
The userId is stored in the Firestore database and is used to identify the user who created the story.

The createdAt and updatedAt fields are timestamps that are automatically generated by the Firestore database when the story is created and updated.
